OnAmbiguousCommand ( ACTION )

A block that describes what to do when a valid transitive or ditransitive action is parsed from the input, but an object is parsed that shares a name with more than one accessible object, based on the current context.

Valid on Elements

WORLD PLAYER

Parameters

ACTION Any ACTION type, OPTIONAL. The action that this block handles.

When it is Called

The OnAmbiguousCommand() block is called when the interpreter can parse a valid action from the interpreted input by name, and the parsed objects have valid names, but they refer to more than one currently accessible object.

When TAME needs to execute this block, it searches for it on the current player, then the world. It will prioritize the blocks that declare a specific action over ones that don't.

Inheritance rules apply when searching for callable blocks, always!

Example

Technical Junk

The OnAmbiguousCommand() block is context sensitive involving the current player. This is assessed when finding the relevant block to execute for the current action being processed. This will affect program flow when players change.
