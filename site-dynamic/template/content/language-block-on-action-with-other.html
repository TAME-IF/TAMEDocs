OnActionWithOther ( ACTION )

A block that describes what to do when an element is the target of a valid ditransitive or transitive action with another object's lineage.

Valid on Elements

WORLD PLAYER ROOM OBJECT

Parameters

ACTION A DITRANSITIVE or TRANSITIVE action. The action that this block handles.

When it is Called

Ditransitive Actions

The onActionWithOther() block is called when a ditransitive action would ordinarily call an onActionWith() or onActionWithAncestor() block, but they are passed over due to no match, and needs to be handled by this entry point.

Transitive Actions

The onActionWithOther() block is called on a room, player, or world when a transitive action needs to handle a target object, but an onAction() block does not exist on the target object, or a matching onActionWith() block didn't exist on the room, player, or world, and then a matching suitable onActionWithAncestor() block didn't exist on the room, player, or world.

Inheritance rules apply when searching for callable blocks, always!

Example

See Also

onAction() — Called to handle an action with another object, or a general action on the current player or room.
onActionWith() — Called by ditransitive or transitive actions for handling an action with another object.
onActionWithAncestor() — Called by ditransitive or transitive actions for handling an action with another non-specific object.
onUnhandledAction() — Called if a command is valid, but an action is unhandled.

Technical Junk

The "first" and "second" objects are determined by the interpreter. The order still matters even if the action is not strict. This stops when it finds a single relevant block.

If not strict, the object test order is always first-onActionWith-second, then second-onActionWith-first. Ancestor checks are the same way.
