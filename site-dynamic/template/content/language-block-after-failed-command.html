AfterFailedCommand ( )

A block that describes what to do after a "failed command" is processed.

Valid on Elements

WORLD

Parameters

No Parameters

When it is Called

The AfterFailedCommand() block on the world element is called after a "failed command" (and the first set of enqueued commands during its execution) are processed. Generally, a "failed command" is a command that is processed that is the result of a bad interpreted action (errors, or "unknown" names).
