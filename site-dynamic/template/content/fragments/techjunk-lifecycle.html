The "lifecycle blocks" (afterSuccessfulCommand(), afterFailedCommand(), and afterEveryCommand()) always run through the list of enqueued commands (added via queueAction(), queueActionObject(), queueActionString(), queueActionObject2(), queueActionForObjectsIn(), queueActionForTaggedObjectsIn()) after the initial handling of the parsed command.

TAME's general flow of handling commands are:

  • Interpret Input.
  • If "successful command",
    • Enqueue Command.
    • Process all Queued Commands.
    • Call afterSuccessfulCommand().
    • Process all Queued Commands.
    Else (failed command),
    • Call appropriate block.
    • Process all Queued Commands.
    • Call afterFailedCommand().
    • Process all Queued Commands.
  • Call afterEveryCommand().
  • Process all Queued Commands.
